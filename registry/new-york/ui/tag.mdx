# Tag

A decorative, **non-semantic** category label using the `--tag-*` palette (9 hues). One hue = one category.

<Canvas of={TagStories.Default} />

## When to use

- Categorising or grouping items by a stable dimension (project, team, topic).
- Read-only labelling where the colour is a mnemonic, not a status.

## When not to use

- **Status or meaning** (success / warning / error) — use a semantic component, not a hue.
- Interactive filters — use `FilterBar`.
- More than ~6 hues on one screen: colour stops being a useful mnemonic.

## Hues

<Canvas of={TagStories.AllHues} />

Keep one hue per category, consistently, across the product.

## Accessibility

Colour is decorative — never the only signal. The text label carries the meaning. The `dot` is purely ornamental. Contrast of label text on its tag background meets WCAG AA via the `--tag-*` tokens.
