# @efounders Design System

A brand-agnostic design-system base, distributed as a **shadcn registry**. Projects consume it with `shadcn add @efounders/...`, then own the code locally. The code is the **single source of truth**; Figma reflects it.

## How this Storybook is organised

- **Get started** — you are here: welcome, installation, contributing.
- **Foundations** — the design tokens (colors, spacing, radius, typography) the whole system is built from.
- **@efounders** — the components this design system *distributes*. These are yours: `Tag`, `Stat`, `FilterBar`, and the `Chat` blocks. **This is the design system.**
- **Base** — the standard shadcn primitives (`Button`, `Input`, `Dialog`…) the project consumes and themes. They are documented upstream by shadcn; the bespoke `@efounders` components compose them.

## The three things that make it AI-native

1. **Tokens by role** — `--primary`, `--muted-foreground`… mapped from a few brand knobs (`--brand-*`, `--radius`, font). Re-skin everything by changing the knobs.
2. **Rules where the agent reads them** — `CLAUDE.md`, `llms.txt`, `docs/COMPONENTS.md`, and a deterministic `ds-check` gate.
3. **Real, reused components** — one canonical component per role, rendered here from the actual source. No hand-made mirror that drifts.

## Source of truth

Code first. Tokens live in `app/globals.css`. Components live in `registry/new-york/` (distributed) and `components/ui/` (base). Never hardcode a value when a token exists — the `ds-check` gate enforces it.
