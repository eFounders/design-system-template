# References

The north-star patterns this product aims for. The tokens say *what things are worth*, the components say *what each one is*, composition says *how they sit on a page*, and this file says *which real-world patterns we are trying to match*.

It holds what the code cannot express: the interaction model and the feel of a pattern, captured as a reference to a real product screen. It is the design direction, frozen in words, so a fresh agent aims at the right target instead of guessing.

## The one rule

**Match the pattern, not the paint.** Borrow the interaction, the density, the flow. Never borrow the reference's colors, type or spacing: render every pattern with *our* tokens and *our* components. A reference is a north star, not a texture to copy. If a reference ever conflicts with the design system, the design system wins.

## How references get here

References come from real products: the **Mobbin MCP** (search real app screens straight from the agent), or plain screenshots the team collects. This file is not a live feed. It holds only the handful the team has *validated* and wants to keep as durable memory. Explore widely, freeze little here.
